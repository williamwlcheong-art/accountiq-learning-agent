## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | none (unicode characters used inline, e.g. &#x2190;, &#x21bb;) |

No framework. No shadcn. This is a vanilla JS/HTML SPA. All UI is hand-authored DOM via `mk()`, `formGroup()`, `styledInput()`, `styledSelect()`, and a new `buildDynamicTable()` helper. Reuse existing CSS classes throughout.

## Screen-by-Screen Interaction Contract

### Screen A: Intake Step 2b — Valuation Advisory Form

**Trigger:** User selects "Valuation Advisory" on step 2 and clicks "Continue".

**Container:** Existing `#wizard-step-2b` card. Existing `#intake-valuation_advisory` div is the root.

**Layout:** Single-column, top-to-bottom. Three visually separated sections within the card, each introduced by an `<h3>` sub-heading using `mk('h3', {style:'font-size:.875rem;font-weight:700;color:var(--navy);margin:1.25rem 0 .75rem'}, 'Section Name')`.

#### Section 1 — Narrative Risk Assessment

Sub-heading: "Narrative Risk Assessment"

Helper text (paragraph below sub-heading): "Describe each risk area in your own words. These inform the qualitative sections of the valuation report."
- Font-size: `.8rem`, color: `var(--muted)`, margin-bottom: `1rem`

Four fields, each built with `formGroup(labelText, styledInput(name, 'textarea', {rows:'3', placeholder:'...'})`:

| Field name (DOM `name` attr) | Label | Placeholder |
|------------------------------|-------|-------------|
| `owner_key_person_dependency` | Owner / Key-Person Dependency | How reliant is the business on the owner or 1–2 key individuals? Would it function without them? |
| `customer_concentration` | Customer Concentration | What % of revenue comes from your top 3 customers? B2B vs consumer split? Contracted vs repeat business? |
| `competitive_position` | Competitive Position & Barriers | Who are the main competitors? What stops new entrants? Any IP, contracts, or brand moat? |
| `growth_strategy_pipeline` | Growth Strategy & Pipeline | How does the business plan to grow? Signed contracts, geographic expansion, new products? |

Field gap: `margin-bottom: 1rem` on each `formGroup` wrapper div.

textarea rows: 3 for all four fields.
textarea resize: not constrained (browser default).

#### Section 2 — Normalisation Schedule

Sub-heading: "Normalisation Schedule"

Helper text: "Pre-filled from your EBITDA add-backs. Edit, remove, or add items specific to this valuation engagement."
- Font-size: `.8rem`, color: `var(--muted)`, margin-bottom: `.75rem`

**Loading state:** Before EBITDA add-backs are fetched, display: `mk('p', {style:'font-size:.8rem;color:var(--muted)'}, 'Loading add-backs...')`. Replace with table once loaded.

**`buildDynamicTable()` function signature:**
```js
buildDynamicTable(container, initialRows)
// container: DOM element (the #intake-valuation_advisory div)
// initialRows: array of {label, amount, rationale} objects pre-filled from ebitda_adjustments
```

**Table structure:**

```
[Label header | Amount ($) header | Rationale header | Actions header]
[text input   | number input      | text input        | [Remove] btn  ]  × N rows
[Add row button spanning full width]
```

Table element: `<table>` with `width:100%;border-collapse:collapse;font-size:.85rem` (matches existing `table` rule).

Column headers: `<th>` with existing `th` CSS rule (`background:#f0f4f8;padding:.6rem .8rem;font-weight:600;font-size:.75rem;color:var(--muted);border-bottom:2px solid var(--border)`).

Columns:
- Label: `<input type="text" name="norm_label_N" placeholder="e.g. Owner salary above market">` — flex: 2
- Amount ($): `<input type="number" name="norm_amount_N" placeholder="0" step="1000">` — width: 120px
- Rationale: `<input type="text" name="norm_rationale_N" placeholder="Brief explanation">` — flex: 2
- Actions: `<button class="btn btn-sm btn-danger" type="button">Remove</button>` — width: 80px

Each cell: `<td>` with `padding:.4rem .8rem`.

Row data attributes: `data-row-index="N"` on each `<tr>` for targeting by Remove handler.

**Add row button:** Full-width secondary button below the table.
```html
<button class="btn btn-secondary" type="button" style="width:100%;margin-top:.5rem">+ Add normalisation item</button>
```
On click: appends a new blank row with `norm_label_{N}`, `norm_amount_{N}`, `norm_rationale_{N}` where N is the current row count.

**Remove row behaviour:** Removes the `<tr>` immediately. No confirmation required (amounts are not destructive — user can re-add). Re-indexes `data-row-index` attributes after removal.

**Empty state:** When all rows are removed, the table body shows one row:
```
[No normalisation items. Click "+ Add normalisation item" to add one.]
```
Displayed as a `<td colspan="4">` with `text-align:center;color:var(--muted);padding:1rem;font-size:.85rem`.

**Answer serialisation:** `wizardSubmitGenerate()` serialises the normalisation table as a JSON array stored under key `normalisations` in the `intake_answers` object:
```js
answers.normalisations = [
  {label: '...', amount: 12000, rationale: '...'},
  ...
]
```
This is handled by a custom collection path in `wizardSubmitGenerate()` — the existing `querySelectorAll('[name]')` loop will capture individual `norm_label_N` / `norm_amount_N` / `norm_rationale_N` inputs, but the executor MUST add a post-processing step to collate them into the `normalisations` array before the `fetch()` call. See Data Collection note below.

### Screen B: Step 3 — Researching/Generating Status

The existing step 3 card is extended to handle the new `researching` status emitted by the backend during agentic research.

**Existing states retained unchanged:**
- `queued` — existing `.alert-success` + "Your report generation has been queued."
- `failed` — existing `.alert-error` + retry button
- `done` — existing `.alert-success` + "Your report is ready!"

**New intermediate status: `researching`**

When `_updateStep3Status()` receives `status === 'researching'`:
```
genDiv visible, failDiv hidden, doneDiv hidden
statusText.textContent = 'Status: researching — gathering market data and WACC inputs...'
```
Additionally show an `.alert.alert-info` above the status text:
```
"The agent is researching your company, sector, and WACC inputs from RBNZ and Damodaran. This typically takes 60–90 seconds."
```
This alert is injected once into `genDiv` on first `researching` status and not repeated on subsequent polls.

**Polling interval:** 3 seconds (unchanged from existing `_startReportPolling`).

### Screen C: Report Viewer — Table Section Rendering

**Context:** The `GET /wizard/report/{id}/view` endpoint returns report content JSON. Sections with tables return:
```json
{
  "narrative": "string",
  "table": {
    "headers": ["Col1", "Col2", ...],
    "rows": [["val", "val", ...], ...]
  }
}
```

**Trigger:** When `_updateStep3Status()` receives `status === 'done'`, a "View Report" button becomes visible in `wiz-step3-done`. Clicking it calls `loadReportViewer(reportId)`.

**`_renderSectionTable(tableObj)` helper:**

```js
function _renderSectionTable(tableObj) {
  // tableObj: {headers: [], rows: [[]]}
  // Returns a <div> containing a <table> using existing table CSS
}
```

Table wrapper div: `margin-top:.75rem;overflow-x:auto` (handles wide WACC/valuation tables on narrow screens).

Table element: `<table>` — uses existing `table`, `th`, `td`, `tr:hover td` rules.

Column headers: `<th>` per `tableObj.headers[i]` — built with `.textContent` (XSS rule AUTH-03).

Rows: one `<tr>` per `tableObj.rows[i]`, one `<td>` per cell — built with `.textContent`.

Numeric cells: right-aligned. Detection rule: if `cell.trim()` starts with a digit, `-`, or `$` then `td.style.textAlign = 'right'`.

First column: left-aligned always (label column in all table sections).

**Section rendering order for valuation_advisory (12 sections):**

Each section rendered as:
```
<div class="profile-section" style="margin-bottom:1.5rem">
  <div class="profile-section-title">{section_key formatted as title case with underscores replaced by spaces}</div>
  <p style="font-size:.875rem;line-height:1.75;color:var(--text)">{narrative text}</p>
  {if section.table exists: _renderSectionTable(section.table)}
</div>
```

Section title formatting: `section_key.replace(/_/g, ' ').replace(/\b\w/g, c => c.toUpperCase())`.

All narrative text and cell values set via `.textContent` — XSS rule AUTH-03 applies.

**Table sections** (render both narrative + table):
- `financial_performance`
- `normalisations_schedule`
- `balance_sheet_summary`
- `wacc_assumptions`
- `valuation_summary`

**Narrative-only sections** (render narrative paragraph only):
- `introduction`
- `business_overview`
- `market_position`
- `valuation_methodology`
- `dcf_analysis`
- `multiples_crosscheck`
- `disclaimer`

**Disclaimer section special treatment:** Render inside a styled container:
```
background:#fff8e1;border:1px solid #f59e0b;border-radius:6px;padding:.75rem 1rem
font-size:.8rem;line-height:1.6;color:#92400e
```
This visually distinguishes the disclaimer from narrative content (matches existing profile-warning banner pattern).

## Data Collection Contract

### Intake answer shape for `valuation_advisory`

The `wizardSubmitGenerate()` function's answer collection must produce the following shape for the `intake_answers` field of `POST /wizard/report/generate`:

```json
{
  "owner_key_person_dependency": "string",
  "customer_concentration": "string",
  "competitive_position": "string",
  "growth_strategy_pipeline": "string",
  "forecast_horizon": "3" | "5",
  "revenue_growth_cagr": 8.0,
  "terminal_growth_rate": 2.5,
  "normalisations": [
    {"label": "Owner salary above market", "amount": 50000, "rationale": "Owner pays self $150k vs market $100k"},
    ...
  ]
}
```

The executor must add a post-processing block in `wizardSubmitGenerate()` specifically for `valuation_advisory`:

```js
if (_wizardReportType === 'valuation_advisory') {
  // Collate individual norm_label_N / norm_amount_N / norm_rationale_N inputs into array
  var norms = [];
  var i = 0;
  while (answers['norm_label_' + i] !== undefined || answers['norm_amount_' + i] !== undefined) {
    var lbl = answers['norm_label_' + i] || '';
    var amt = answers['norm_amount_' + i] || 0;
    var rat = answers['norm_rationale_' + i] || '';
    delete answers['norm_label_' + i];
    delete answers['norm_amount_' + i];
    delete answers['norm_rationale_' + i];
    if (lbl) norms.push({label: lbl, amount: parseFloat(amt) || 0, rationale: rat});
    i++;
  }
  answers.normalisations = norms;
}
```

## Validation Contract
