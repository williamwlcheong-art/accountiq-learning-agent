## 2026-07-19 methodology correction for PR 3A

The earlier decision to research WACC during report generation is superseded for authoritative paid valuation inputs. New valuations select exactly one active adviser-approved WACC assumption set before checkout, and freeze its full values and approval metadata in snapshot schema 2. Research may still support an adviser creating a future assumption-set version, but it cannot alter a paid valuation after checkout.

PR 3A also requires complete FCFF assumptions before persistence or any external call. Same-period D&A and approved operating NWC components support safe ratio prefills. Capex remains an explicit customer confirmation unless an adviser-approved schedule is supplied. Aggregate investing cash flow is never treated as capex. Confirmed zero remains distinct from missing and needs a rationale.

## Area 1: Questionnaire redesign

| Question | Options presented | Selected |
|----------|-------------------|----------|
| What should the intake capture? | Narrative risk inputs only / Financial assumptions only / Both | **Both** |
| Narrative areas to cover | Owner dependency / Customer concentration / Competitive position / Growth strategy | **All four** |
| Financial section: what user provides vs. extracted | Forecast assumptions only / Normalisations + forecast / Full WACC inputs + forecast | **Normalisations + forecast** |
| Normalisation table approach | Confirm + adjust existing only / Confirm + add new / Separate normalisation table | **Separate normalisation table** (pre-filled from Phase 3, fully editable, add new items) |

**Rationale:** The 23-question scoring model is being dropped because it produces a score-driven multiple, not the qualitative risk narrative needed for a Propellerhead-style report. The hybrid approach captures what the agent can't research (owner dependency, customer detail, growth plans) and what the financials don't specify (normalisation items, forecast assumptions).

## Area 2: WACC & DCF model

| Question | Options presented | Selected |
|----------|-------------------|----------|
| Drop EV/EBITDA multiples? | DCF-only / Keep both / Multiples as cross-check | **Multiples as cross-check** |
| WACC determination | Agent researches / User enters, agent validates / Hardcoded bands | **Agent researches WACC inputs** (historical decision, superseded for authoritative paid valuation inputs) |
| WACC type | Real WACC (inflation-adjusted) / Nominal WACC with explicit terminal growth | **Nominal WACC, user enters terminal growth** |
| Illiquidity discount | Keep Damodaran formula / Simplified fixed discount / Drop it | **Keep Damodaran formula** |

**Rationale:** DCF is the professional standard for NZ SME valuations; multiples as a cross-check adds credibility without driving the conclusion. The historical agent-researched WACC decision removed a technical burden from business owners. The current paid-valuation authority is an active, adviser-approved, versioned WACC set selected and frozen before checkout. Nominal WACC was chosen for simplicity.

## PVM-08 correction decisions

**Date:** 2026-07-18

| Question | Selected |
|----------|----------|
| Snapshot authority | PR #15 makes upload revisions and statement-period authority immutable; PR #16 freezes paid report inputs before checkout. Both are merged. |
| PR 2A scope | Typed valuation inputs and the EV-to-equity bridge only. |
| Units | Accept `whole`, `thousands`, and `millions`, then convert to whole currency units. Fail closed for unsupported units. |
| Currency | Require one well-formed ISO-style currency code across selected inputs. Do not convert currencies implicitly. |
| Periods | Accept annual year, FY year, and annual year-end date forms. Use the latest complete P&L period and a compatible balance-sheet period. Fail closed on ambiguity or incompatibility. |
| EBITDA hierarchy | Use reported EBITDA first, including zero. Otherwise use same-period EBIT plus depreciation and amortisation. Do not use NPAT plus depreciation. |
| Debt | Include only clearly interest-bearing approved categories: bank debt and loans, overdrafts, finance leases, hire purchase, and shareholder or director loans. Fail closed on ambiguous combined liabilities. |
| Cash | Deduct only clearly unrestricted cash. Restricted cash and cash held in trust require clarification. |
| Surplus assets | Default to zero. Include an amount only when explicitly approved with a rationale and source. |
| Equity bridge | Equity value equals enterprise value minus interest-bearing debt plus unrestricted cash plus approved surplus assets. |
| FCFF | Originally named PR 2B; now sequenced as PR 3B, the Decimal FCFF engine, and still required before live UAT. |

**Rationale:** Financial arithmetic must consume explicit, traceable values. Ambiguous units, currencies, periods, and balance-sheet classifications stop checkout for clarification rather than producing a plausible but unsupported valuation.
