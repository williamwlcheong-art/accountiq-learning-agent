## Task 1 Detail: Frontend Changes

### Sections Removed
- Legacy 23-question EV/EBITDA scoring form with 8 category groups (History, Financial, People, Suppliers, Customers, Premises, Competition, Growth/Opportunities)
- `valCategories` array with 23 scored questions
- Sector dropdown (for multiples scoring)
- DCF Inputs (WACC) manual entry section: risk_free_rate, equity_market_risk_premium, beta, cost_of_debt_pretax, corp_tax_rate, weight_equity, revenue_growth_rate, terminal_growth_rate (old decimal form), forecast_years

### Sections Added

#### New buildDynamicTable() Helper
Added inside `_buildIntakeForms()` below `buildSimpleForm`, co-located with all DOM helpers.
- Generates add/remove-capable table rows with `norm_label_N`, `norm_amount_N`, `norm_rationale_N` inputs
- Empty state: shows message "No normalisation items. Click '+ Add normalisation item' to add one."
- Remove Item button deletes row and refreshes empty state
- Add button appends blank row with auto-incremented index

#### New Valuation Advisory Intake (Three-Section Form)

**Section 1: Narrative Risk Assessment**
- 4 textareas: `owner_key_person_dependency`, `customer_concentration`, `competitive_position`, `growth_strategy_pipeline`
- All using `styledInput(name, 'textarea', {rows:'3', placeholder:...})`

**Section 2: Normalisation Schedule**
- Async pre-fill from `GET /wizard/company/{id}/ebitda-adjustments` (wizard-scoped, Plan 03 Task 4)
- Loading indicator while fetch is in progress
- Falls back to empty table on fetch error
- `buildDynamicTable(normWrap, initialRows)` called after successful fetch

**Section 3: Financial Assumptions**
- Forecast horizon radio group (3 years / 5 years), `name="forecast_horizon"`
- Revenue growth CAGR number input, `name="revenue_growth_cagr"`, 0-100 range
- Terminal growth rate number input, `name="terminal_growth_rate"`, 0-20 range

### Normalisations Collation (wizardSubmitGenerate)
Inserted before the fetch() call, valuation_advisory branch only:
1. Validates `forecast_horizon` is 3 or 5
2. Validates `revenue_growth_cagr` is a number between 0 and 100
3. Validates `terminal_growth_rate` is a number between 0 and 20
4. Collates `norm_label_N` / `norm_amount_N` / `norm_rationale_N` indexed inputs into `answers.normalisations = [{label, amount, rationale}, ...]`
5. Removes indexed keys from answers object (clean JSON to backend)

### Researching Status State (_updateStep3Status)
- Added `mk()` re-declaration inside `_updateStep3Status()` (needed since `mk` is scoped to `_buildIntakeForms`)
- New `researching` branch: injects `.alert.alert-info` div (`#wiz-researching-alert`) once on first poll
- Status text: "Status: researching — gathering market data and WACC inputs…"
- Alert text: "The agent is researching your company, sector, and WACC inputs from RBNZ and Damodaran. This typically takes 60–90 seconds."
- Alert injected only once (checks `document.getElementById('wiz-researching-alert')`)
