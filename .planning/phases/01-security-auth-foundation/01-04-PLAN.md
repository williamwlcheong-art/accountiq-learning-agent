JS API contract (functions to add):
- initApp(): page-load entry. Calls /auth/me. 200 -> showMainApp(user); 401 -> showAuthWall(false).
- showAuthWall(expired): displays #auth-page; if expired===true, shows session-expired alert.
- showMainApp(user): hides #auth-page, shows #main-app, sets #user-email textContent, shows #user-header, calls showPage('dashboard').
- toggleAuthForm(): swaps #login-form / #register-form display.
- submitLogin(): reads #auth-email, #auth-password; POSTs to /auth/login; on success calls showMainApp; on error shows alert per UI-SPEC copy.
- submitRegister(): reads #reg-email, #reg-password, #reg-password-confirm; validates; POSTs /auth/register; on success calls showMainApp.
- doLogout(): POSTs /auth/logout; clears cookies on response; calls showAuthWall(false).
- loadAccount(): fetches /auth/me; populates #acct-email and #acct-created textContent.

Modified existing helpers:
- apiFetch(path): add `credentials: 'include'`; on 401 response call showAuthWall(true).
- apiPost(path, fd): add `credentials: 'include'`; on 401 call showAuthWall(true).

      const pw = document.getElementById('auth-password').value;

        fd.append('password', pw);

          showMainApp(user);
          return;

          _setAuthAlert(detail, 'error');
        }
      } catch (e) {

      } finally {
        btn.disabled = false;

          showMainApp(user);
          return;
        }

          let detail = 'Password must be at least 8 characters.';

          _setAuthAlert(detail, 'error');
        } else {
          let detail = 'Could not create account.';
          try { detail = (await res.json()).detail || detail; } catch (e) {}
          _setAuthAlert(detail, 'error');
        }
      } catch (e) {
        _setAuthAlert('Connection error. Check your network and try again.', 'error');
      } finally {
        btn.disabled = false;
        btn.textContent = originalText;
      }
    }

    async function doLogout() {
      try {
        await fetch(`${API}/auth/logout`, { method: 'POST', credentials: 'include' });
      } catch (e) { /* ignore — we're logging out either way */ }
      showAuthWall(false);

    async function loadAccount() {
      const user = await apiFetch('/auth/me');
      if (!user) return;  // 401 already triggers showAuthWall via apiFetch
      document.getElementById('acct-email').textContent = user.email;
      // Format created_at "May 2026" — input is SQLite datetime('now') format like "2026-05-05 12:34:56"
      let created = '—';
      if (user.created_at) {

        if (!isNaN(d)) {
          created = d.toLocaleString(undefined, { month: 'long', year: 'numeric' });
        }
      }
      document.getElementById('acct-created').textContent = created;
    }

      const user = await apiFetch('/auth/me');

        // apiFetch already called showAuthWall(true) on 401 — but on first load it should be 401 with no "expired" framing
        // Override the framing for first load:
        showAuthWall(false);
        return;
      }
      showMainApp(user);
    }

    // Init — entrypoint
    initApp();

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser DOM -> backend cookie | Auth credentials only flow via HttpOnly cookie set by /auth/login or /auth/register; JS never reads the cookie |
| Auth-wall form -> /auth/* | Form values cross the localhost network boundary; HTTPS not used in dev (per D-CONTEXT) but cookies do not leak via XSS due to HttpOnly |
| 401 from any protected route -> client | Triggers showAuthWall(true) — does not reveal which user the failed call belonged to |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | XSS injecting JS to steal session cookie | mitigate | Cookie is HttpOnly (Plan 03 set_cookie); XSS surface itself reduced to zero in Plan 02; viewNarrative re-verified in Task 4 manual smoke |
| T-04-02 | Information Disclosure | Auth wall leaks app data to unauthenticated user | mitigate | #main-app starts hidden; initApp gates the reveal on /auth/me 200; no data fetched until showMainApp runs |
| T-04-03 | Spoofing | CSRF — third-party site triggers POST /companies with user's cookie | mitigate | SameSite=Lax cookie (Plan 03) + CORS allowlist of localhost only (Plan 02) — both layers prevent the cookie from being sent on a cross-site form POST |
| T-04-04 | Information Disclosure | Password value retained in form between sessions | mitigate | showAuthWall clears auth-password / reg-password / reg-password-confirm input values on display; password fields use type="password" (browser autofill respects user policy) |
| T-04-05 | Denial of Service | Submit button double-click during async login | mitigate | Submit buttons disabled while request is in flight; re-enabled in finally |
| T-04-06 | Tampering | Credentials sent on cross-origin requests after browser cookie attack | accept | Same-origin only deployment in v1; documented in CONTEXT; production hardening (secure cookie, HTTPS-only) deferred to deployment phase |
| T-04-07 | Information Disclosure | First-time-load briefly shows "Session expired" message | accept | Cosmetic flicker — initApp overrides with showAuthWall(false) within one event loop tick; documented as acceptable in Task 3 action notes |
