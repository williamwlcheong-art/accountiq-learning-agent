```python
async def _register_admin(client, email="admin@example.com", password="correcthorse"):
    """Register a user as admin by patching the module-level OWNER_EMAIL constant."""
    import auth as _auth_module
    original = _auth_module.OWNER_EMAIL
    _auth_module.OWNER_EMAIL = email.lower()
    try:
        r = await client.post(
            "/auth/register",
            data={"email": email, "password": password},
        )
    finally:
        _auth_module.OWNER_EMAIL = original
    return r
```

| Boundary | Description |
|----------|-------------|

| Environment → auth.py module | OWNER_EMAIL read from .env at startup; if unset, no auto-admin is granted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-5-01 | Elevation of Privilege | `register` route — OWNER_EMAIL comparison | mitigate | Both sides lowercased at load/input time (`OWNER_EMAIL = ...strip().lower()`, `email = email.strip().lower()` at line 110); case-mismatch attack surface eliminated |
| T-03-5-02 | Elevation of Privilege | `require_admin` — dependency chain | mitigate | `require_admin` chains `Depends(get_current_user)` — never re-decodes JWT; unauthenticated callers receive 401 (not 403); confirmed by PATTERNS.md |
| T-03-5-03 | Elevation of Privilege | OWNER_EMAIL unset edge case | accept | If `OWNER_EMAIL` is empty string, the guard `if OWNER_EMAIL and email == OWNER_EMAIL` short-circuits — no account auto-receives admin; documented in .env.example |
| T-03-5-04 | Information Disclosure | `require_admin` returns 403 (not 404) | accept | Admin route existence is not sensitive (documented exception to Phase 2 404 policy in CONTEXT.md — 403 is the correct semantic per success criterion 3) |
